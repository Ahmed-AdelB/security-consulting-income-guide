## 13. Recon & Asset Discovery Pipeline

### 13.1 Recon Fundamentals
- Enumerate subdomains and infrastructure assets within scope.
- Identify new acquisitions and regional domains (ccTLDs).
- Use passive DNS and certificate transparency to track new assets.

### 13.2 EU-Specific Recon Notes
- Many EU firms use localized subdomains (fr, de, nl, es).
- GDPR-related endpoints often reveal high-impact data flows.
- Region-specific portals may have weaker access control.

### 13.3 Change Detection
- Monitor asset changes daily or weekly.
- Track new API endpoints and staging environments.
- Alert on new logins, partner portals, or admin panels.

### 13.4 Recon Workflow Example
1. Asset enumeration (subdomains, IP ranges, app IDs).
2. Live host validation and tech fingerprinting.
3. Content discovery and endpoint mapping.
4. Authentication and role mapping for logic testing.
5. Continuous monitoring for new assets.

## 14. Vulnerability Classes with High ROI

### 14.1 Access Control & Authorization
- IDORs and broken object-level authorization.
- Role escalation and privilege misconfiguration.
- Cross-tenant data exposure in multi-tenant apps.

### 14.2 Authentication & Session
- Authentication bypass via logic flaws or weak verification.
- Session fixation and session management issues.
- MFA bypass or recovery flow abuse.

### 14.3 Injection & Execution
- SQL injection or ORM injection in legacy systems.
- Command injection or deserialization flaws.
- SSRF leading to internal service access.

### 14.4 Business Logic
- Payment or subscription logic bypass.
- Coupon, discount, or credit abuse.
- Workflow abuse in onboarding or KYC flows.

### 14.5 Data Exposure & Privacy
- Unprotected PII access or misconfigured storage buckets.
- Insecure direct object references in document systems.
- Data export features lacking authorization checks.

## 26. Templates & Checklists

### 26.1 Report Template (Copy/Paste)
- Title: [Severity] Short impact summary
- Summary: One-paragraph overview of the issue and affected assets
- Steps to Reproduce: Numbered list with exact requests and parameters
- PoC: Script, payload, or video link
- Impact: Business impact and attacker outcomes
- Remediation: Suggested fix or mitigation

### 26.2 Pre-Submission Checklist
- Scope confirmed against program policy
- Vulnerability reproduced on in-scope asset
- Impact verified and documented
- Evidence captured (screenshots, logs, requests)
- Duplicate check against prior reports or known issues

### 26.3 Post-Submission Checklist
- Monitor for triage questions
- Respond within 24-48 hours
- Provide additional evidence if requested
- Document outcome and lessons learned

### 26.4 Recon Checklist
- Enumerate subdomains and public assets
- Identify authentication flows and role boundaries
- Map sensitive endpoints and file upload paths
- Test authorization across user roles
- Review API documentation for hidden parameters

## 28. Appendix: Glossary
