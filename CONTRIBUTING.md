# Contributing to Security Consulting Income Guide

Thank you for your interest in contributing! This guide is community-driven and benefits from real-world experiences.

## How to Contribute

### 1. Report Issues

Found outdated information? Platform changed their rates? Let us know:

- Open an issue with the `outdated` label
- Include the file path and specific section
- Provide a source for the correction if available

### 2. Add Platform Information

Have experience with a platform not covered? We'd love to add it:

1. Fork the repository
2. Create a new file in the appropriate `01-platforms/` subdirectory
3. Follow the existing template format
4. Include:
   - Platform overview
   - Application process
   - Rate ranges (cite sources)
   - Pros and cons
   - Tips for success
5. Submit a pull request

### 3. Share Income Data

Anonymous income reports help everyone. Share:

- Your specialization (DFIR, GRC, pentest, etc.)
- Years of experience
- Geographic region
- Platform(s) used
- Monthly/annual income range
- Hours worked

We'll aggregate this data while keeping individual contributions anonymous.

### 4. Improve Existing Guides

- Fix typos and formatting
- Add missing information
- Update outdated rates
- Improve clarity

## Content Guidelines

### DO:

- Cite sources when providing statistics
- Share personal experience (anonymized if preferred)
- Include both pros and cons of platforms
- Provide actionable advice
- Be specific about rates, requirements, and processes

### DON'T:

- Include personal identifying information
- Share confidential client information
- Make unsubstantiated claims
- Provide legal or financial advice
- Include affiliate links without disclosure

## File Format

All guides should use this structure:

```markdown
# Platform/Topic Name

## Overview

Brief description (2-3 sentences)

## Key Stats

- Rate range: $X-$Y/hr
- Volume: Low/Medium/High
- Time to first payment: X weeks

## How It Works

Step-by-step process

## Requirements

- Minimum experience
- Certifications helpful
- Application process

## Pros and Cons

### Pros

- Bullet points

### Cons

- Bullet points

## Tips for Success

Actionable advice from experience

## Resources

- Official links
- Community discussions
- Related guides in this repo

## Last Updated

YYYY-MM-DD
```

## Pull Request Process

1. Ensure your changes follow the format guidelines
2. Update the "Last Updated" date in modified files
3. Add yourself to CONTRIBUTORS.md (optional)
4. Submit PR with a clear description of changes
5. Respond to any review feedback

## Code of Conduct

- Be respectful and constructive
- Focus on helping others succeed
- No gatekeeping or discouraging newcomers
- Share knowledge freely

## Questions?

Open an issue with the `question` label and we'll respond promptly.
